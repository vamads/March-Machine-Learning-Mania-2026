# 🏀 March Madness Ensemble 2026

![Project Banner](banner.png)

## Overview
This repository contains a state-of-the-art stacking ensemble designed to predict the outcomes of the **2026 Kaggle March Machine Learning Mania** tournament. The system combines multiple specialized machine learning models to capture team efficiency, graph-based strength of schedule, and historical momentum.

## 🏗 Architecture

The ensemble utilizes a two-level stacking architecture designed to optimize for **Log Loss**:

### Level 0: Base Models
1.  **Tabular Model (XGBoost/LightGBM)**: Processes detailed regular season box scores to capture efficiency, possession-based metrics, and "Madness Metrics" (variance and upset potential).
2.  **Graph Model (GNN)**: Uses Graph Neural Networks (GraphSAGE/GCN) to model the transitive nature of college basketball, learning embeddings that represent team strength relative to their full schedule.
3.  **Elo Model (LightGBM)**: Focuses on historical performance and win/loss trajectories.
4.  **Raddar-Cauchy Model (Model E)**: A Cauchy-loss regressor that specializes in predicting point differentials and leaf-node stacking for final probabilities.

### Level 1: Meta-Learner
The final predictions are generated by an **XGBoost Meta-Learner** that combines base model outputs with interaction features:
*   **Consensus Spread**: Captures ensemble uncertainty.
*   **Algo Conflict**: Measures disagreement between tabular and Elo projections.
*   **Injury & HCA Sensitivity**: Adjusts predictions based on late-season player availability and historical home-court advantage impacts.

## 🛠 Features & Pipelines

### 📊 Tabular Pipeline
- **Four Factors**: eFG%, TOV%, ORB%, and FTR.
- **Madness Metrics**: Captures "Live-by-the-3" reliance and defensive chaos creation.
- **Symmetric Doubling**: Ensures the model is perfectly balanced by training on both A vs B and B vs A variations.

### 🕸 Graph Pipeline
- **Dynamic Weighting**: Edges are weighted by possession-adjusted efficiency margins and game location (Home/Away/Neutral).
- **Time Decay**: Recent games are given higher weight to capture end-of-season improvement.

### 📈 Ordinals & Momentum
- **System Filtering**: Focuses on top-tier predictors like KenPom (POM), Sagarin (SAG), and Torvik (TOR).
- **Momentum Slope**: Calculates the "Trend" between Selection Sunday and 30 days prior to capture team trajectory.

---

## ⚖️ Evaluation & Inference
- **Optimization**: The model is strictly optimized for **Log Loss**.
- **Temporal Firewall**: Training uses a strict Day 132 boundary to prevent future-data leakage from the tournament itself.
- **Calibration**: Uses Platt Scaling to translate raw model scores into well-calibrated tournament probabilities.
